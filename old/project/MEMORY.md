## 2026-02-20 — Улучшения: удаление приёмов, покрытие базы продуктов, промпты, статистика

### Что сделано

**1. Удаление приёмов пищи (soft-delete)**
- Добавлено поле `deletedAt DateTime?` в модель `Meal` (миграция `20260220100000_add_meal_deleted_at`).
- При просмотре «Сегодня»/«Вчера» под сводкой появляются inline-кнопки `✕ Название (ккал)`.
- Нажатие → подтверждение → soft-delete → пересчёт сводки в том же сообщении.
- Удаление доступно только для сегодня и вчера. Удалённые приёмы исключены из всех агрегатов.
- Callback-маршрут: `del:<id>` → `delc:yes|no:<id>`. Новый обработчик `handleMealDeleteCallback`.

**2. Снижение частоты «выдумка ИИ»**
- `scripts/import-off.ts`: дефолт страниц с 50 → 200; добавлен импорт по категориям OFF (20 категорий: beverages, coffees, dairy, snacks, sauces и т.д.). Ожидаемый результат: 10–20k продуктов в FoodReference вместо ~5k.
- `src/fsm/handlers.ts` `estimateDraft`: перед вызовом LLM `buildFoodReference` — пробует OFF text search для макс. 2 отсутствующих продуктов. Найденные сохраняются в FoodReference, исключаются из LLM-батча.
- Важно: OFF text search добавляет ~7с первый раз на продукт, затем мгновенно из кеша.

**3. Промпты LLM — диапазоны веса**
- Добавлены типичные порции (капучино ~200 мл, эспрессо ~30 мл, тарелка супа ~300 мл и т.д.).
- Правило ширины диапазона: стандартные напитки/штучные → ±10–15%; домашние блюда без чёткого размера → ±25–40%.
- `parseMeal` теперь принимает `portionHints` и рендерит `{{portion_hints}}` в шаблоне.

**4. Исправления статистики и средних**
- **Timezone bug:** `dayKey` теперь сохраняется в часовом поясе пользователя (был UTC). Новый хелпер `toDayKeyForUser(date, timezone)` в `src/utils/time.ts` через `Intl.DateTimeFormat`.
- **Период «месяц»:** изменён с «последних 30 дней» на «текущий календарный месяц» (с 1-го числа до вчера). Метка — реальное название месяца на русском («Февраль», «Март» и т.д.).
- **Период «неделя»:** метка «7 дн.» → «Неделя».
- **Гейтинг:** «Всё время» появляется от 7 дней истории (было 30), месяц — от 1 дня в текущем месяце (было 7).
- **Отображение:** «X дн. с данными» теперь inline в заголовке, а не отдельной строкой внизу.

### Важные ограничения
- OFF text search в `estimateDraft` — макс. 2 продукта за запрос, т.к. каждый добавляет ~7с задержку.
- seed-aliases.ts намеренно НЕ расширялся — владелец против ручных алиасов, предпочитает реальные базы.
- Soft-delete сохраняет исторические данные в БД (принцип immutability).

## 2026-02-20 — Модель продаж и подписка (SALES_MODEL)

### Что сделано
- **Единая подписка:** триал 14 дней (с первого подтверждения) → бесплатный тир (3 оценки, 1 подтверждение в день) или Premium. Одна дата окончания доступа: `User.subscribedUntil` (конец последнего дня). При оплате во время триала: оставшиеся дни триала + 30 дней добавляются к дате окончания.
- **Админ:** `/admin setend <userId> YYYY-MM-DD` — выставить любую дату окончания подписки; `/admin grant` обновляет `subscribedUntil` (не уменьшает); `/admin revoke` сбрасывает подписку. Все действия админа пишутся в `SubscriptionAdminLog` (отдельная история).
- **Напоминания:** за 7, 3 и 1 день до конца триала — сообщение об ограничении и Premium; в день окончания триала — «Триал закончился». Вечернее напоминание только для Premium/триала. Напоминание и «Статистика» (неделя/месяц) — только при полном доступе.
- **Гейтинг по SALES_MODEL:** «Почему так?», «Сегодня», «Вчера» — всем; лимит оценок/подтверждений и проверка `canEstimate` в `estimateDraft`; Статистика и напоминание — только `hasFullAccess`.
- Миграция: `SubscriptionAdminLog`, поля User для напоминаний о триале (`trialReminderSent7At`, `trialReminderSent3At`, `trialReminderSent1At`, `trialEndReminderSentAt`). `endOfDay()` в `utils/time` для «последний день включительно».

Источник: SALES_MODEL.md, запрос владельца.
