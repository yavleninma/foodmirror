# DECISIONS

Ключевые архитектурные решения и контекст.

## 2026-03-02 — Rewrite v2.0 на монорепо

- **Решение:** Полный rewrite на `apps/api + apps/web + packages/shared` с TypeScript.
- **Почему:** Прототип v1 не масштабировался (один файл, без типизации, без нормального деплоя).
- **Альтернатива:** инкрементальный рефакторинг v1.
- **Принято:** командой проекта.

## 2026-03-02 — Express + Prisma + PostgreSQL

- **Решение:** API на Express 5, данные через Prisma и PostgreSQL.
- **Почему:** простая поддержка, миграции и type-safe запросы.
- **Альтернатива:** JSON-хранилище, Fastify/Hono.
- **Принято:** командой проекта.

## 2026-03-02 — Деплой на Railway

- **Решение:** Railway как основная платформа (web + postgres).
- **Почему:** быстрый запуск, низкая операционная сложность.
- **Альтернатива:** VPS/Fly/Vercel.
- **Принято:** командой проекта.

## 2026-03-02 — Telegram initData в заголовке

- **Решение:** Telegram auth-подпись передаётся через `X-Init-Data`.
- **Почему:** единообразно для GET/POST, не смешивается с payload.
- **Альтернатива:** query/body.
- **Принято:** командой проекта.

## 2026-03-02 — Guest-mode вне Telegram

- **Решение:** вне Telegram разрешён только базовый сценарий `POST /api/insight`.
- **Почему:** можно попробовать продукт в браузере без потери фокуса на Mini App.
- **Ограничение:** личные функции (`history`, `stats`, `user/*`) доступны только в Telegram режиме.
- **Принято:** командой проекта.
