## Как вносить изменения

### Перед изменением
1. Прочитай `project/ARCHITECTURE_MAP.md` — пойми где находишься
2. Прочитай `project/DOMAIN_BIBLE.md` — пойми правила домена
3. Проверь `project/DECISION_LOG.md` — может быть уже обсуждалось

### После изменения
1. Обнови `project/MEMORY.md` если узнал что-то важное
2. Обнови `project/DECISION_LOG.md` если принял решение
3. Убедись что код компилируется (`npm run build`)

### Стиль коммитов
```
<тип>: <краткое описание>

Типы: feat, fix, refactor, docs, test, chore
Примеры:
  feat: add daily stats comparison
  fix: protect unmentioned weights in corrections
  refactor: extract override parsing from handlers
  docs: update ARCHITECTURE_MAP
```
