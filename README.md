# FoodMirror — Core Direction v2.0

## 0. Контекст

В папке [`old/`](./old/) находится предыдущая версия проекта — Telegram-бот для подсчёта калорий.

Этот код можно:

- использовать как инфраструктуру,
- переиспользовать модули,
- заимствовать логику хранения данных,

**НО** философия старой версии не должна определять новую.

- Старая версия = **трекер**.
- Новая версия = **поведенческий инструмент**.

---

## 1. Что мы строим

**FoodMirror** — поведенческий инструмент для офисных и IT-людей, которые не хотят считать калории, но хотят не «расползаться».

- Это **Telegram Mini App**.
- Вне Telegram есть **guest-режим** для базового сценария (фото → инсайт), без личной статистики.
- Это не бот-анкета. Не счётчик БЖУ. Не диета.

Цепочка: **Фото → Короткий инсайт → Микро-коррекция → Еда.**

---

## 2. Для кого

**Целевая аудитория:** офис / IT, 25–40 лет.

**Проблема:** сидячая работа, перекусы, булки к кофе, жидкие калории, вечерние компенсационные ужины, постепенный набор веса.

**Не хотят:** считать, диету, сложный трекинг, фитнес-лекции.

**Хотят:** контроль, осознание, простой инструмент перед едой.

---

## 3. Главная идея

Мы не считаем калории. Мы боремся с **автоматизмом**.

Каждое фото должно:

- выявить главный поведенческий риск,
- дать одну корректировку,
- занять не больше 5–10 секунд внимания.

Если ответ длиннее — это ошибка.

---

## 4. Что обязательно в первой версии (MVP)

- Telegram Mini App интерфейс
- Приём фото еды
- Быстрый AI-анализ
- Короткий структурированный ответ (вердикт + 1 правка)
- Сохранение истории фото
- Выбор цели пользователя («не расползтись»)
- Статистика: фото за неделю, стрик

**Ничего больше.**

---

## 5. Что НЕ входит в первую версию

- Социальные функции
- Чат между пользователями
- Рецепты
- Графики ради графиков
- Подробный подсчёт БЖУ
- Геймификация ради геймификации
- Медицинские рекомендации

**Если возникает сомнение — не добавлять.**

---

## 6. Главная метрика

Не вес. Не MAU. Не калории.

**Главная метрика:** среднее количество фото **до еды** на пользователя в неделю.

Если люди перестают фоткать — продукт не работает.

---

## 7. Техническое направление

### Стек

- **Платформа:** Telegram Mini App
- **Фронтенд:** React 19 + TypeScript + Vite 6 + Tailwind CSS 4
- **Бэкенд:** Express 5 + TypeScript + Prisma 6
- **База данных:** PostgreSQL
- **AI:** OpenAI GPT-4o-mini (vision)
- **Деплой:** Railway
- **Монорепо:** npm workspaces

### Структура

```
apps/api/      — бэкенд (Express + TypeScript + Prisma)
apps/web/      — фронтенд (React + Vite + Tailwind)
packages/shared/ — общие типы и константы
ai-system/     — документация проекта, решения, архитектура
```

### Быстрый старт

**Подробная инструкция:** [`LOCAL_SETUP.md`](./LOCAL_SETUP.md)

```bash
# 1. PostgreSQL (Docker)
docker run -d -p 5432:5432 -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=postgres -e POSTGRES_DB=foodmirror --name foodmirror-db postgres:16-alpine

# 2. .env
cp .env.example .env
# Заполнить: TELEGRAM_BOT_TOKEN, OPENAI_API_KEY, DATABASE_URL=postgresql://postgres:postgres@localhost:5432/foodmirror

# 3. Запуск
npm install && npm run db:push
npm run dev:api   # терминал 1 — бэкенд :3000
npm run dev:web   # терминал 2 — фронт :5173
```

Деплой: [`ai-system/DEPLOY.md`](./ai-system/DEPLOY.md)

## 8. Отношение к `old/`

**Разрешено:** использовать базу, переиспользовать авторизацию, хранилище, заимствовать инфраструктуру.

**Запрещено:** переносить философию «считать всё», переносить интерфейс трекера, делать продукт вокруг цифр.

---

## 9. Долгосрочная цель

Через 3 месяца пользователь:

- автоматически фотографирует еду,
- видит риски до анализа,
- уменьшает порцию до инсайта,
- чувствует контроль без подсчёта.

Если продукт стал внутренним зеркалом — он успешен.

---

## 10. Принцип разработки

Строим поведенческий инструмент, а не инженерный шедевр.

Каждая новая функция должна отвечать на вопрос: **увеличит ли это количество фото до еды?**

Если нет — не внедрять.
